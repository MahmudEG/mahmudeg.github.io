## Hi, I'm Mahmud

I'm a systems engineer focused on Microsoft hybrid infrastructure — the point where on-premises Active Directory and Windows Server meet Azure and Microsoft Entra ID, and everything has to keep running while it gets more secure.

Day to day I work across Active Directory and Windows Server — design, Group Policy, hardening, and tiered administration — hybrid identity with Entra Connect and Conditional Access, and the security controls that close the gaps attackers actually use. My background is in telecommunications: I studied Electrical and Electronic Engineering at the University of Tripoli, where I went deep on networking and cybersecurity. That grounding still shows up in how I think about resilient, well-segmented systems.

## What this blog is

This is where I document what I build and troubleshoot, in production and in the lab: hands-on walkthroughs, real-world fixes, and condensed study notes. No vendor-brochure summaries — every post is meant to be reproducible, with the actual commands, configurations, and trade-offs. If you work in Windows Server, Active Directory, Azure, or Entra ID — or you're studying for the same certifications — you'll find labs you can follow step by step.

The topics I write about most: hybrid identity, Active Directory security and hardening, Windows Server, Azure and Entra ID, Conditional Access and Zero Trust, with the occasional networking or CTF write-up.
