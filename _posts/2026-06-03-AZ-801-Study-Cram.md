## 1. Secure Windows Server On-Premises and Hybrid Infrastructures (25–30%)

### OS Security
- **Exploit Protection** — configure via Windows Security Center or GPO / `Set-ProcessMitigation`
- **Windows Defender Application Control (WDAC)** — controls which apps/scripts can run; replaces AppLocker for modern workloads
- **Credential Guard** — isolates LSASS secrets using VBS; requires UEFI + Secure Boot
- **SmartScreen** — filters phishing/malicious downloads
- **OSConfig** — enforces Windows Server security baselines declaratively
- **Windows LAPS** — stores local admin passwords in Entra ID or AD DS; rotates automatically

### Hybrid Active Directory Security
- **Microsoft Entra Password Protection** — bans weak passwords on-prem by syncing policy from Entra ID
- **Protected Users group** — cannot use NTLM, Kerberos delegation, or RC4 encryption
- **RODC** — Password Replication Policy (PRP) controls what credentials are cached
- **Authentication Policy Silos** — restrict which accounts can log into which hosts
- **Microsoft Defender for Identity (MDI)** — detects lateral movement, pass-the-hash, golden ticket attacks
- **Audit and disable NTLM** — GPO: `Network Security: Restrict NTLM`

### Azure Security Integration
- **Microsoft Sentinel** — ingest Windows events via AMA (Azure Monitor Agent)
- **Defender for Cloud** — security posture score + threat alerts for hybrid servers
- **Defender for Servers Plan 2** — adds MDE integration, file integrity monitoring, vulnerability assessment

### Network & Storage Security
- **Windows Defender Firewall** — manage via GPO or Windows Admin Center
- **Domain Isolation** — IPsec to require auth between domain members
- **NSGs** — stateful rules on Azure VM NIC or subnet
- **BitLocker** — TPM-based; store recovery keys in AD DS or Entra ID
- **Azure Disk Encryption** — uses BitLocker; keys in Azure Key Vault

## 2. Implement and Manage Windows Server High Availability (15–20%)

### Failover Clustering
- Supports on-premises, Azure IaaS, and hybrid (stretch clusters)
- **Workgroup clusters** — no AD DS required (DMZ use case)
- **Stretch cluster** — spans two sites; uses Storage Replica for sync/async replication
- **Storage Spaces Direct (S2D)** — hyperconverged storage using local disks across nodes
- **Azure Witness (Cloud Witness)** — uses Azure Blob Storage; recommended over file share witness
- **Network ATC** — automates adapter teaming and RDMA config for clusters
- **Floating IP** — needed in Azure clusters (no multicast); configure via load balancer probe

### Quorum
- Even nodes → add a witness. Odd nodes → witness optional.
- Options: Node Majority, Node + Disk, Node + File Share, Cloud Witness

### Storage Spaces Direct
- Pool NVMe/SSD/HDD across nodes into software-defined storage
- Resiliency options: two-way mirror, three-way mirror, parity, nested resiliency
- Upgrade nodes via rolling upgrade (one at a time)

## 3. Implement Disaster Recovery (10–15%)

### Azure Backup
- **Recovery Services Vault** — configure backup policies (frequency, retention)
- **Azure Backup Server (MABS)** — on-prem backup for VMs, SQL, SharePoint
- **Instant restore** — restore from snapshot before transferring to vault
- **Restore options:** File recovery, restore to new VM, restore encrypted VM (needs Key Vault access)

### Azure Site Recovery (ASR)
- **Replication:** On-prem Hyper-V/VMware → Azure, or Azure VM → secondary region
- **Network mapping** — map source vNet to target vNet
- **Recovery plan** — ordered VM group failover; supports Automation runbooks
- **Replication policy** — RPO threshold, retention, app-consistent snapshot frequency
- **Test failover** — validates DR plan without impacting production

### Hyper-V Replica
- Primary → Replica host (on-prem to on-prem)
- Frequencies: 30 seconds, 5 minutes, 15 minutes
- Supports extended replication (3 sites)
- Failover types: **Test** (non-disruptive), **Planned** (zero data loss), **Unplanned** (last synced state)

> 💡 Know **RTO vs RPO** — it drives technology choice (Failover Cluster vs ASR vs Hyper-V Replica).

## 4. Migrate Servers and Workloads (20–25%)

### Storage Migration Service (SMS)
- Migrates files, shares, permissions, local users/groups
- Cut over moves identity (IP, name) to new server transparently
- Can target: Azure VMs or Azure File Shares
- Orchestrated via Windows Admin Center

### Azure Migrate
- **Appliance** — lightweight VM for agentless discovery and assessment
- **Migration types:** Agentless (Hyper-V, VMware), Agent-based (physical servers)
- After migration: install Azure VM agent, configure backup, test connectivity

### Windows Server Version Upgrades
- **In-place upgrade** — supported across consecutive versions (e.g. 2016 → 2019 → 2022 → 2025)
- **DHCP** — `Export-DhcpServer` / `Import-DhcpServer`
- **Print server** — `printbrm.exe`
- **IIS** — Web Deploy or IIS Migration Tool; check app pools, bindings, modules
- **RDS** — migrate session hosts, licensing servers, connection brokers

### IIS to Azure
- **Azure App Service** — use App Service Migration Assistant for compatibility check
- **Containers** — Windows containers for AKS or Azure Container Apps

### AD Forest Migration
- **ADMT** — migrates users, groups, computers between forests
- Raise schema and functional levels when upgrading to Server 2025

## 5. Monitor and Troubleshoot Windows Server Environments (15–20%)

### Monitoring Tools

| Tool | Use Case |
|---|---|
| Performance Monitor | Real-time counters, Data Collector Sets |
| Windows Admin Center | GUI monitoring and alerting |
| System Insights | ML-based predictive analytics (disk, CPU, memory, network) |
| Azure Monitor / DCR | Centralized monitoring via AMA; alerts and dashboards |
| VM Insights | Process/dependency mapping on Azure VMs |

### Troubleshooting Cheat Sheet

| Problem | First Steps |
|---|---|
| Connectivity | `Test-NetConnection`, check NSG + Windows Firewall rules |
| Name resolution | `nslookup`, `Resolve-DnsName`, check DNS zones |
| Windows Update | Check WU logs, WSUS connectivity |
| Time sync | `w32tm /query /status`, check NTP / PDC emulator |
| Boot failures | WinRE, `bootrec /fixmbr`, `bootrec /rebuildbcd` |
| Disk encryption | Check TPM, Key Vault access, `manage-bde` |
| AD replication | `repadmin /showrepl`, `repadmin /replsummary` |

### Active Directory Recovery
- **AD Recycle Bin** — restore deleted objects; enable with `Enable-ADOptionalFeature`
- **DSRM** — offline AD database repair and restore
- **Repadmin** — diagnose and fix AD replication issues
